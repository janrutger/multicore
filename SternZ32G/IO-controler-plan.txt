we gaan de Z32 familie uitbreiden met een IO-contoller chip
Zodat het SternZ32 platform oren en ogen krijgt


De IO-Controler is de brug tussen de CPU en de periherels, zoals:
- Karakterbased display
- Toetsenbord
- Grafisch 480x640 display

Om de InmosZ32 CPU met de IO-controler te laten praten breiden we de CPU instructie set uit
- OUT Rx reg#       Rx is het register met de output waarde reg# is het registernummer van de IO controler
- IOSYNC            Non-blokking tick voor de controller, hij verstuurd wat er verstuurd moet worden, en kijkt of er een input waaarde (kbd) klaar staat
- IN Rx reg#        leest de wrde uit io-controller reg# en schrijft deze in Rx, dit is NULL als niemand een toets heeft ingedrukt.

De IO controler ondersteund een Read-flag en een write-flag zodat de CPU instucties kunnen zien of ze geldig aan de beurt zijn, anders stallen


Voorbeeld toetsenbord:
    Iedere toets aanslag wordt naar de IO-controler gestuurd en daar gebufferd.
    Bij een IOSYNC controleerd de read sectie of de read-flag op 0 staat (de vorige read is door de CPU verwerkt)
    dan haalt hij 1 waarde uit her KBD buffer en zet deze in het aangewezen output register en zet de read flag op 1

    Als de CPU een IN Rx reg# instructie uitvoerd, kopiierd de CPU de waarde uit reg# naar Rx en zet deflag op 0


Voorbeeld display
    Met de OUT Rx reg# instructie vullen we de IO-registers die het instructie blok vormen, kijk eerst of de write-flag op 0 staat.
    De IO-contoller heeft een write command-blok met:
        device type         karrakter           Grafisch
        value1              char-to-print       color-to-plot
        vaule2              -                   X-as
        value3              -                   y-ax
        value4              -                   -                   (who knows)
        instruction         print               plot                (clear, move_cursor ....)

        Door in het insstructie register te schrijven wordt de write-flag op 1 gezet

    IOSYNC controleerd of er write werk te doen valt, leet het commandblok en stuurd de inhoudt naar het device
    en zet de flag weer op 0


    Op de achtergrond op het mainboard in SternZ32 krijgen de devices die betaan een tick.
    tijdens de tick worden de informatie verwerkt (output device buffert de input stream) 
    het toetsenbord device kijkt bij een tick of er fysieke KBD aanslagen zijn geweest, en zet deze in het buffer van de IO-controleer.



    Periferals <> IO Controller <> CPU


    devices
        Er zijn 3 periferal devices een 24x80 karakter display, een 480x640 grafisch display en een toetsenbord
        De devices zijn gebassserd op Tkinter voor de intergratie met de fysieke hardware

        de display devices hebben eigen display-memory en leunen niet op main memory
        De display devices hebben meerdere instructies
        Karacter device:
            - print
            - clear             Clear screen
            - setcursor
        Grafisch
            - plot
            - clear             Clear screen
        
        Karakter device doet line-wrapping wanneer de regel lengte van 80 wordt overschreden en doet screenscrolling wanneer het aantal van 24 regels wordt overschreden
        het grafische device vangen we de clipping op door wardes buiten bereik te negeren en weg tegooien
    
    IO Controller
        Voor de communicatie met de CPU heeft de IO-controler de volgende registers
        -   reg0        devicetype/ID   1=karacter 2=Grafisch
        -   reg1        value1          char to print, of color to print
        -   reg2        value2          X-coordinate
        -   reg3        value3          Y-coordinate
        -   reg4        value4          Gereserveerd voor later
        -   reg5        instruction     1=print, 2=plot, 3=clear, 4=setcursor
                                        writing to reg5 triggers the write-flag to 1
        -   reg6        KBD-value1      reading from reg6 resets the read-flag to 0

    CPU(InmosZ32)
        De volgens CPU instructies zijn nodig
        OUT Rx reg#         Controleert het write-flag
                            als write-flag = 0, Schrijf Rx naar regx
                            als wirte-flag = 1, Stall de CPU

        IOSYNC              write sectie:
                                leest de command registers
                                Write-flag = 1, stuur de inforamtie naar het device, zet write-flag = 0
                                Write-flag = 0, doe niks

                            read sectie
                                Leest het hardware keyboard buffer
                                als read-flag = 0 en er input beschikbaar is schrijf eerste waarde naar reg6
                                zet de read-flag op 1

                                Als er geen data beschikbaar is in het hardware buffer, schrijf 0 (null) naar reg6 en read-flag op 0

        IN Rx reg6          Leest reg6 en schrijft dezze naar Rx
                            zet de read-flag naar 0

                            (progammmeur moet zelf de inhoud van Rx controleren en testen op null)
